## 💡 My Approach to Problem Solving

Every problem I solve follows a consistent framework:

```
1. Understand   →  Read carefully, identify input/output constraints
2. Examples     →  Trace through given examples by hand
3. Brute Force  →  Think of the naive O(n²) or O(n³) solution first
4. Optimize     →  Identify the bottleneck, apply the right pattern
5. Code         →  Write clean, readable code with edge case handling
6. Complexity   →  Always note Time and Space complexity
```

This approach has helped me move from guessing to *pattern recognition* — the real skill in DSA interviews.
